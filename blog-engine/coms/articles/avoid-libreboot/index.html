Why you should be weary of Libreboot

1. Merge with osboot, now allows running non-free firmware

On November 15th 2022, libreboot merged with it's sister project, "osboot", which allowed non-free firmware.
Since then libreboot has basically become a coreboot installer with less hardware support.

2. Libreboot now requires you install spyware microcode updates.

Along with the osboot merge, libreboot now forces you to install the latest microcode updates for your processor,
if it wasn't bad enough that it allowed the use of non-free firmware,
now it forces you to use proprietary microcode updates.

What do I use now?

The only real solution now is modifying the libreboot source code yourself to remove the need to use proprietary firmware.
However I am currently working on a coreboot distribution like libreboot pre-osboot merge.
